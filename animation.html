animation: it automatically allow the changing effect on any element. For example the animation word is changing color in main hading.
Youtube Video Link....

Syntax:
There are 2 ways to define the working of animation.
 
 @keyframes name_of_animation
  { 
    form {css property at starting }
    to {css property at end}
  }
 or 
 @keyframes name_of_animation
  { 
    0% {css property at starting }
    20% {css property at 20% of animation}
    40% {css property at 40% of animation}
	.
	.
	.
	.
    100% {css property at end}
	
  }


Different properties of the animation are:

1. animation-name:name_of_animation; (it is the name of animation as we given in @keyframens)

2. animation-duration:2s; (it is the time how long the animation take.)

3. animation-timing-function:ease;
It has following 9 values:
S.no. Unit Description
1. ease slow start,then fast,then end slowly
2. linear same speed from start to end
3. ease-in slow start
4. ease-out slow end
5. ease-in-out slow start and slow end
6. step-start equivalent to steps(1,start)
7. step-end Equivalent to steps(1,end)
8. steps(#,start/end)
9. cubic-bezier lets you define your own values(0 to 1)


4. animation-delay:2s; (after how much time does the animation start)

5. animation-iteration-count:5; (how many time does animation run, it can be given in number. If we want to run it infinite time we use word "infinite")

6. animation-direction:alternate;
It has following 4 values:
S.no. Unit Description
1. normal default
2. reverse start from 100%(end) and end at 0%(start)
3. alternate start from 0%(start) move to 100%(end) and end at 0%(start)
4. alternate-reverse start from 100%(end) move to 0%(start) and end at 100%(end)


7. animation-fill-mode:backwards;
It has following 4 values:
S.no. Unit Description
1. none default value
2. backwards already at 0% and start from here and end at initially
3. forwards start at initially and end at 100% and pause here
4. both both backward and forward works i.e. already at 0% and start from here and end at 100% and pause here


8. animation-play-state:paused;
It has only 2 values:
1. paused (it pause the animation on any condition such as we hover on that element)
2. running (it run the animation on any condition such as we hover on that element)

9. animation:arun 2s ease-in 3s infinite alternate backwards paused; (it is the shorthand of above all propertie of the animation)
The order of giving value is:
name_animation duration timing-function delay iteration-count direction fill-mode play-state.
